06-34. Three explicit invocation forms cover all `From`/`Into` impls: `x.into::[U]()`, `Into::into(x)`, and `From::from(x)`. (¬ß7.8)

06-36. The trait-path form works only when the target type is inferable from context: `let x: f64 = Into::into(5_i32)`; otherwise the turbofish method form is used. (¬ß7.8)
06-37. On failure, `?` propagates the failure value up the call stack, terminating the enclosing function early. (¬ß7.9)
06-38. At a `?` site the source's error type must equal the enclosing function's error type or be convertible to it via `From`. (¬ß7.9)
06-39. `expr?` evaluates to the source's success type at that site, bound locally or used inline. (¬ß7.9)
06-40. `?` imposes no constraint between the source's success type and the enclosing function's return success type. (¬ß7.9)

06-44. The error-type rule is the only relationship `?` enforces: no implicit success-type coercion and no fallback through other trait machinery. (¬ß7.9)
06-45. Failure handling uses a two-track model ‚Äî traps versus typed values ‚Äî chosen at the operation site when code is written. (¬ß8)
06-46. A failure encoded on one track cannot be silently converted to the other. (¬ß8)

06-60. A non-exhaustive `match` is a compile error, never a runtime trap. (¬ß8.1)

06-64. A trap never becomes a `Result::Err` value. (¬ß8.1)
06-65. A `Result::Err` never aborts the program. (¬ß8.1)
06-66. There is no `try`/`catch` mechanism for traps. (¬ß8.1)
06-67. `panic` is a prelude built-in available unqualified in every scope. (¬ß8.2.1)

06-78. `Option` and `Result` are ordinary stdlib generic enums with no language-level special-casing of their identity. (¬ß8.3)
06-79. `Option[T]` is the stdlib enum with variants `Some(T)` and `None`. (¬ß8.3)
