### Reactive-cell type-model overhaul · streams §13.18 + effects §13.19 (ruled 2026-06-14 · applied · cold-reviewed · findings struck)
The streams+effects 5b sub-pass, taken as a foundational type-model overhaul rather than per-finding patches.
- **One umbrella.** `Cell[T]` is the **sole** umbrella over reactive cells; `Signal[T]` is **demoted** from umbrella to a concrete host/placement-written value cell. `Readable[T]`/`Writable[T]` removed.
- **Per-kind value types.** `derived` → `Derived[T]` (no history), `recurrent[N]` → `Recurrent[T, N]` (N-deep self-history); `Derived[T]` is the degenerate N=0 case. A reactive value expression produces `Derived[T]`.
- **Ruling A.** Value-reading operator/function parameters are typed `Cell[T]` (a stream has no current value, so it is excluded at the read site, not the signature); computed value outputs are `Derived[T]` (§13.2.8, §13.17.3; new entry 029-129).
- **Recurrent streams are distinct types** `RecurrentRingStream[T,B,H]` / `RecurrentGateStream[T,B,H]`; plain ring/gate are the H=0 degenerate cases. `.past`/`.previous` are producing-context only (F119).
- **`Sink[T]` and the `sink` keyword removed entirely** (§13.18.4 deleted, pipe Case 3 removed): events are streams, so a stream's write side is mechanism (a `= source`, or the host API), never a named type — making §13.2.7 ("programs describe, never write") hold without exception. (Supersedes F141; folds F142/F145.)
- **Effects are flat I/O surfaces.** `desired:` holds `derived`/`recurrent` (values) and `stream … = source` (events the effect feeds from its own parameters/observed cells); `observed:` holds `signal`/`stream`. `recurrent` is **allowed in `desired:`**, forbidden in `observed:`. A desired stream's source may read the effect's own observed cells (**feedback**) provided the cycle passes a delay/commit boundary. **No exposed sinks**, no external `|>` into effect cells, and **no `repeat`/topology** in effects (per-element I/O is a node's `effects:`-clause repeat).
- **No capacity elision** (F116): concrete stream types always carry capacity (`const N` generic for polymorphism). **Observation cells are `Derived[T]`** (runtime-computed, §13.18.6).

### StreamPolicy overhaul — streams §13.18 re-typed + the 12-finding close (ruled 2026-06-17 · applied · cold-reviewed · findings struck)
Resolving F123 exposed that the whole stream-typing model was wrong the same way value cells had been: policy "inheritance" was prose, capacity was hung on the umbrella `Stream`, and the concretes were ad-hoc. Re-typed streams and closed the rest of the streams/effects findings in one coordinated pass.
- **`Stream[T, P]` with `P: StreamPolicy`.** A stream's type is its element `T` plus a **policy** `P`; the policy carries its own buffer parameters, so **capacity belongs to the policy, not the umbrella `Stream`** (a future unbounded policy carries none). `StreamPolicy` is a **language-sealed** trait (like `Into`/`TryInto`); its members are `Ring[N]`, `Gate[N]`, `RecurrentRing[B,H]`, `RecurrentGate[B,H]`. The erased supertype `Stream[T]` is kept (concretes widen, zero-cost).
- **`RingStream[T,N]` etc. become transparent generic aliases** of `Stream[T, Ring[N]]` — which required adding **generic type aliases** to §4.2 (the alias feature was concrete-only). That kept ~80 of 97 type-name sites untouched.
- **Preserve = thread `P`** (type-true, in the signature — not a "compiler propagates" rule): `map`/`filter`/`skip`/`take`/`throttle`/`scan`/`pairwise` thread `P`; value-returning ops (`to_signal`/`count`/`fold`/`any`/`all`) take erased `Stream[T]`; `merge`/`merge_gate` name a concrete policy; `where` threads its LHS `P` (**F123**).
- **F118**: recurrent self-history `H` is a **separate** allocation from buffer `B` (`RecurrentRing[B,H]`); the `+N` default-capacity rule dropped. **F122**: `throttle` clock = nanoseconds, `Cell[u64]`. **F120**: `dropped_total` per-stream. **F121**: shrinking input `.past k` retains the newest `k`. **F124**: an effect needs ≥1 block (the block-less `log` example rewritten with a `desired:` block; params don't auto-bind). **F125**: "promote to operator" composes the effect through a pipe — it does not host it. **F146**: `stream` permitted in operator bodies.
- **F143 / F117**: **no implicit `signal`→`stream` conversion** — `to_stream` replaced by explicit `to_ring_stream`/`to_gate_stream`; a stream source must contain ≥1 stream; signals in a stream expression are **sampled** (not emitted), so they add no capacity — which makes the §13.18.2 and `where` (§13.18.10.3) capacity rules agree (**F148**). A zero-stream (signals-only) `stream` binding is now a compile error (F117), consistent with §13.18.7.3. **F115** confirmed moot (the overhaul had already removed `Writable[T]`).

Findings struck: **F115, F117, F118, F120, F121, F122, F123, F124, F125** (full ledger 70 → 61; tally 30 amb · 24 inc · 6 con · 1 unsound) and **F143, F146, F148** (Session-4 block 3 → 0, block now empty).
